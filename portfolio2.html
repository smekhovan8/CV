Утилита для работы с датчиком профилей

ООО "Эйдос-Робототехника", июнь 2017

    Датчик профилей (лазерный триангуляционный 2D сканер) — это устройство, содержащее в себе лазер, проецирующий прямую линию на контролируемую поверхность, и камеру, детектирующую отражённую световую линию. Этот тип датчика, помимо прочего, предназначен для установки на роботизированных сварочных комплексах и служит для коррекции траектории сварки с учётом небольших отклонений геометрии сварного соединения, а также для мониторинга параметров и контроля качества сварных соединений. Целью этой работы было создание утилиты, позволяющей пользователю максимально просто работать со сканером, разрабатываемым компанией Эйдос-Робототехника.

    Реализованное Windows-приложение позволяет настраивать параметры камеры (время экспозиции, гамма-коррекцию и т.д.), выполнять внутреннюю калибровку сканера в ручном и полуавтоматическом режимах, проводить мониторинг параметров сварных соединений различных типов.

    В рамках данного проекта мной были разработаны и реализованы следующие алгоритмы:

        1. поиск полосы лазера на изображении в виде набора профилей с возможностью фильтрации нежелательных бликов;

        2. вычисление коэффициентов дисторсии линз с дальнейшей её компенсацией;

        3. вычисление матрицы проективного преобразования координат из плоскости изображения в плоскость лазера;

        4. поиск точек сварки на полученных профилях для сварных соединений четырёх различных видов.

Кроме этого мной была разработана удобная обёртка для API камеры (использовалась промышленная камера Basler), реализующая методы подключения, установки параметров, захвата изображения и отключения, а также весь пользовательский интерфейс приложения.

    Подробнее о данном проекте можно прочитать на странице компании.

Инструменты разработки: C++, Qt, OpenCV, Ceres Solver, Visual Studio.

Симулятор 7-осевого робота KUKA LBR iiwa

ООО "Эйдос-Робототехника", ноябрь 2016

    LBR iiwa — это лёгкий промышленный робот компании KUKA, предназначенный для коллаборативной работы с человеком. Целью этой работы было создание симулятора, позволяющего отладить все движения робота на этапе программирования. Это позволяет в случае ошибки не допустить коллизий дорогостоящего манипулятора с окружающими предметами или с самим собой.

    Разработанное приложение позволяет производить трёхмерную визуализацию перемещения манипулятора KUKA LBR iiwa. Виртуальная модель робота выполняет команды на движение, полученные либо по сети, либо непосредственно через пользовательский интерфейс. В процессе движения выполняется проверка на самоколлизии и на выход углов шарнирных соединений за допустимые диапазоны.

    В этом проекте мной были разработаны и реализованы следующие алгоритмы:

        1. аналитические методы расчёта прямой и обратной кинематики робота с сохранением заданной конфигурации;

        2. методы выполнения различных типов движения (joint, point-to-point, linear) с поддержанием заданной скорости.

Приложение создано в среде Unity 3D на языке C#. Методы расчёта кинематики вызываются из отдельной DLL, написанной на C++.

    Данная разработка была внедрена на Казанском Электротехническом Заводе (КЭТЗ).

Инструменты разработки: C++, C#, MATLAB, Eigen, Visual Studio, Unity 3D.

Генерация траектории перемещения манипулятора для лазерной очистки

ООО "Эйдос-Робототехника", июнь 2016

    Лазерная технология очистки применяется для удаления с металлических поверхностей ржавчины, лакокрасочных покрытий, подготовки поверхности под покраску и т.д. Установка аппаратуры лазерной очистки на роботизированный манипулятор позволяет автоматизировать этот процесс. Идею можно лучше понять из этого видео. Цель этого проекта в использовании 3D сканера для получения трёхмерной модели очищаемой поверхности и автоматической генерации траектории движения манипулятора, которая позволила бы произвести очистку. При этом накладывается ограничение на максимально возможную ширину луча лазера.

    В этом проекте мной был разработан алгоритм, предназначенный для построения плоской траектории перемещения манипулятора на основе имеющейся 3D модели изделия.

Инструменты разработки: MATLAB.

Вычисление TCP инструмента

ООО "Эйдос-Робототехника", февраль 2016

    Для того, чтобы роботизированный манипулятор мог точно оперировать установленным на него инструментом, он должен знать положение этого инструмента (т.н. Tool Center Point или TCP) относительно своего фланца. Иначе говоря, должна быть определена матрица перехода из системы координат инструмента в систему координат фланца. Цель этой работы была в создании простой утилиты, позволяющей выполнять вычисление такого преобразования (калибровку TCP).

    Для калибровки TCP существует алгоритм, основанный на многократном касании кончиком инструмента фиксированной (опорной) точки при различных положениях фланца и регистрация всех этих положений. По набору полученных положений методом наименьших квадратов возможно вычислить требуемую матрицу перехода.

    Разработанная мной программа реализует существующий алгоритм. Программа представляет собой простое консольное приложение, написанное на C++ с применением библиотеки линейной алгебры Eigen. При запуске она загружает данные с положениями фланца из заранее подготовленного файла и выводит результат расчёта на экран.

Инструменты разработки: C++, MATLAB, Eigen, Visual Studio.

Расчёт координат шарниров гексапода

ООО "Эйдос-Робототехника", октябрь 2015

    Про кинематическую схему робота Гексапод и необходимость методов расчёта прямой и обратной кинематики было рассказано ранее. Для управления этим роботом также необходимо точно знать координаты точек крепления штанг к основанию и к подвижной платформе (точки Ai и Bi соответственно на изображении платформы Стюарта ниже). Вследствие неидеальной сборки конструкции эти координаты немного отличаются от заданных в CAD модели. Целью этой работы было создание методики и разработка приложения для уточнения координат шарниров Гексапода.

    В соответствие с разработанной методикой на каждую штангу крепился специальный маркер, который обнаруживался высокоточным лазерным интерферометром. Далее штангу вручную перемещали сначала за нижний конец (отсоединив её от платформы), а затем за верхний конец (отсоединив её от основания). В процессе перемещения регистрировались координаты маркера. В итоге для каждой из шести штанг было получено по два облака точек — для шарниров основания и подвижной платформы. Каждое такое облако представляло собой набор точек, который приближённо аппроксимировался сферой. Координаты центров найденных сфер и были искомыми координатами шарниров.

    Мной было разработано приложение, позволяющее по найденным облакам точек вычислять координаты шарниров платформы и основания, центрировать их и выравнивать в горизонтальной плоскости. Приложение было написано на C#. Для аппроксимации облаков точек сферической поверхностью и вычисления координат центров использовалась библиотека AlgLib. Для трёхмерной визуализации использовалась библиотека OpenTK.

Инструменты разработки: C#, AlgLib, OpenTK, Visual Studio.
